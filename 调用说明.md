## 🎯 模型拦截功能

**当前配置：** 无论客户端请求什么模型，都会被修改为 `openai/gpt-3.5-turbo`

### 修改为其他模型

编辑文件：`vim /home/XXBAi/HKUST/router/rules/request.yaml`

找到这一行：
```yaml
model: "openai/gpt-3.5-turbo"
```

修改为你想要的模型，例如：
```yaml
# GPT-4 Turbo
model: "openai/gpt-4-turbo"

# Claude 3 Haiku
model: "anthropic/claude-3-haiku"

# Claude 3 Sonnet
model: "anthropic/claude-3-sonnet"

# Gemini Pro
model: "google/gemini-pro"

# Llama 3 70B
model: "meta-llama/llama-3-70b-instruct"
```

保存后重载规则（无需重启）：
```bash
curl -X POST http://你的IP:8765/rules/reload
```

## 📡 调用方式

### 方式 1: Python OpenAI SDK

```python
from openai import OpenAI

# 注意：base_url 指向你的服务器，不需要 API Key（服务器已配置）
client = OpenAI(
    api_key="not-needed",  # 或者任意字符串
    base_url="http://你的服务器IP:8765/v1"
)

# 即使这里写 gpt-4，实际也会被规则改成 openai/gpt-3.5-turbo
response = client.chat.completions.create(
    model="gpt-4",  # ← 会被拦截修改
    messages=[
        {"role": "system", "content": "You are helpful."},
        {"role": "user", "content": "Hello, who are you?"}
    ]
)

print(response.choices[0].message.content)
```

### 方式 2: cURL

```bash
# 调用代理
curl -X POST http://你的 IP:8765/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "任何模型名都会被修改",
    "messages": [
      {"role": "user", "content": "Hello!"}
    ]
  }'
```

### 方式 3: HTTP 请求 (任意语言)

```http
POST http://你的 IP:8765/v1/chat/completions
Content-Type: application/json

{
  "model": "gpt-4",
  "messages": [{"role": "user", "content": "Hello!"}]
}
```

### 方式 4: Claude Code 接入

```bash
# 设置环境变量
export ANTHROPIC_BASE_URL=http://你的 IP:8765

# 或者在配置文件中指定
```

## 🔧 禁用/启用模型拦截

**临时禁用拦截**（允许客户端指定模型）：

编辑 `rules/request.yaml`:
```yaml
- id: "force_model_openrouter"
  enabled: false  # ← 改为 false
  ...
```

重载规则：
```bash
curl -X POST http://你的 IP:8765/rules/reload
```

## 🧪 测试示例

### 1. 发送请求（使用虚假模型名）

```bash
curl -X POST http://你的 IP:8765/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "super-ai-god-model",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

### 2. 查看 Trace，确认模型被修改

```bash
curl http://你的 IP:8765/traces?limit=1 | python3 -m json.tool
```

你会看到：
- `request.raw.model` = `"super-ai-god-model"` (原始)
- `request.modified.model` = `"openai/gpt-3.5-turbo"` (修改后)

## 🆘 故障排除

**Q: 请求返回 401 错误**
A: 检查 `.env` 中的 `OPENAI_API_KEY` 是否正确配置

**Q: 模型没有被修改**
A:
1. 检查 `rules/request.yaml` 中 `enabled: true`
2. 运行 `curl -X POST http://你的 IP:8765/rules/reload`
3. 查看 Trace 确认规则是否命中

**Q: 无法访问服务器**
A: 确保服务器防火墙允许 8765 端口访问
